# Architecture

This app is a frontend-only Vite React game. There is no backend. The browser calls OpenAI directly with `VITE_OPENAI_API_KEY`, so this is a local development workflow, not a production-safe deployment model.

## Runtime Flow

1. `PromptEntry` collects the world prompt.
2. `useGameRun.submitPrompt` calls `generateScenario`.
3. `generateScenario` asks the chat model for structured JSON, normalizes array lengths, validates the schema, and returns a `Scenario`.
4. `BriefingScreen` shows the mission statement.
5. `useGameRun.beginExploration` renders the starting room with `generateRoom`.
6. `GameView` displays the Pannellum panorama and movement controls.
7. On each step, `useGameRun.step` computes the next coordinate, assigns a room catalog index, renders that room if needed, and updates run status.

## State Ownership

- `GameRun` is the source of truth for the current play session.
- `visited` maps `coordKey(coord)` to generated `RoomData`.
- `coordToCatalogIndex` maps each coordinate to the assigned scenario catalog entry.
- `prebuiltRooms` is legacy-compatible state. Current room rendering is coordinate-aware and should not rely on pregenerated unplaced rooms.
- `roomsReady` is a lightweight loading counter, currently used for the starting room.

## Coordinate Model

Coordinates are grid positions:

- Start: `[0, 0]`
- Destination: generated by `pickDestinationCoord(seed, optimalSteps)`
- Distance: Manhattan distance
- Progress: `gradientPosition(coord, start, destination)`

Direction movement:

```ts
N => [x, y - 1]
S => [x, y + 1]
E => [x + 1, y]
W => [x - 1, y]
```

## Room Assignment

Room catalog index assignment is intentionally stable:

- Start coordinate gets index `0`.
- Destination coordinate gets index `31`.
- Intermediate coordinates get unused middle indices near their gradient progress.

This keeps visual room progression roughly aligned with distance to the goal while allowing player wandering.

## Rendering

`Pannellum.tsx` owns the viewer lifecycle:

- Destroys old viewer instances when image data changes.
- Reports yaw to React for heading and movement controls.
- Converts touchpad two-finger wheel gestures into yaw and pitch changes.

Do not put gameplay decision logic in the viewer wrapper. Keep it in `useGameRun`, `coordinates`, or `panorama`.
